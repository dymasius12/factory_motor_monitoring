Data Flow & Architecture

System Architecture

The system uses a microservices architecture with message queues for asynchronous communication between components.

Data Flow:

1. Data Collection

Motor sensors HTTP POST to Node.js API

2. Alert Detection

Node.js validates data and checks thresholds Publishes alerts to RabbitMQ

3. Alert Processing

Python service consumes alerts Stores in PostgreSQL Publishes notifications

4. Visualization

React dashboard subscribes to notifications Displays real-time alerts and charts

Key Integration Points:

  • HTTP API for sensor data ingestion
  • RabbitMQ exchanges for message passing
  • PostgreSQL for persistent storage
  • WebSocket/polling for dashboard updates

Architecture Benefits:

Scalability
Fault Tolerance
Loose Coupling
Real-time Updates
