Python Alert Processing & Database

Alert Processor

The Python alert processor consumes messages from RabbitMQ, processes them, and stores alerts in PostgreSQL.

Key Features:

Consumes from motor.alerts
Stores in PostgreSQL
Publishes notifications
Environment config

Sample Code:

# Process a single alert message
def process_alert(self, ch, method, properties, body):
  try:
    # Parse the alert message
    alert_data = self.parse_alert_message(body.decode('utf-8'))
    
    # Insert alert into database
    alert_id = self.db_manager.insert_alert(
      motor_id=alert_data['motor_id'],
      sensor_type=alert_data['sensor_type'],
      value=alert_data['value'],
      alert_type=alert_data['alert_type']
    )
    
    # Publish notification
    self.publish_notification(alert_data, alert_id)
  except Exception as e:
    logger.error(f"Failed to process alert: {e}")

Database Schema

CREATE TABLE motor_alerts (
  id SERIAL PRIMARY KEY,
  motor_id TEXT NOT NULL,
  sensor_type TEXT NOT NULL,
  timestamp TIMESTAMPTZ NOT NULL,
  value NUMERIC NOT NULL,
  alert_type TEXT NOT NULL
);

SQL Queries

Recent alerts:

SELECT * FROM motor_alerts
WHERE motor_id = :motorId
ORDER BY timestamp DESC
LIMIT 5;

Daily counts:

SELECT
  motor_id,
  DATE(timestamp) as date,
  COUNT(*) as count
FROM motor_alerts
GROUP BY motor_id, date;
