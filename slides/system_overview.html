System Overview

Factory Motor Health Monitoring

Problem Statement:

Automotive assembly plant needs a modern maintenance platform to monitor motor health through vibration and temperature sensors.

Objectives:

• Ingest sensor data in real-time
• Evaluate readings against thresholds
• Flag and process alerts
• Store alerts in database
• Provide monitoring dashboard

Key Components:

1. Data Ingestion

Node.js service with HTTP API

2. Message Queue

RabbitMQ for async messaging

3. Alert Processing

Python service for alerts

4. Database

PostgreSQL for storage

5. Dashboard

React frontend visualization

Monitoring Thresholds:

Vibration: > 2.5 g Temperature: > 80 °C
